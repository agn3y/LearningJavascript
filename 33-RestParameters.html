Rest Parameters

In JavaScript, rest parameters allow a function to accept an indefinite number of arguments as an array. This feature is useful when the number of inputs to a function is not fixed.

Syntax


function myFunction(...args) {
    console.log(args); // Logs an array of all arguments
}

Key Points

  • Rest parameters must be the last parameter in the function definition.
  • A function can have other parameters before the rest parameter.

Examples

1. Collecting Arguments


function sum(...numbers) {
    let total = 0;
    for (let num of numbers) {
        total += num;
    }
    return total;
}

console.log(sum(1, 2, 3)); // 6
console.log(sum(10, 20, 30, 40)); // 100

Here, numbers is an array containing all the arguments passed to the function.

2. Using Rest with Other Parameters


function greet(greeting, ...names) {
    return `${greeting} ${names.join(', ')}`;
}

console.log(greet("Hello", "Alice", "Bob", "Charlie"));
// Output: "Hello Alice, Bob, Charlie"

In this case, greeting captures the first argument, and names collects the rest as an array.

3. Combining Rest with Destructuring


function display(first, second, ...others) {
    console.log("First:", first);
    console.log("Second:", second);
    console.log("Others:", others);
}

display(1, 2, 3, 4, 5);
// First: 1
// Second: 2
// Others: [3, 4, 5]

Difference Between Rest Parameters and Arguments Object

  1. arguments Object:
    • A special object available in non-arrow functions.
    • Contains all arguments as an array-like object but does not support array methods.
    
function example() {
    console.log(arguments);
}
example(1, 2, 3); // [1, 2, 3]
  2. Rest Parameters:
    • Provide an actual array of arguments.
    • Work in both regular and arrow functions.
    • Easier and more modern alternative.
    
const example = (...args) => {
    console.log(args); // An array
};
example(1, 2, 3); // [1, 2, 3]

Use Cases

  • Handling variable-length function arguments.
  • Writing flexible functions like sum or greet.
  • Simplifying functions instead of using the arguments object.

Best Practices

  • Always use rest parameters instead of arguments for new code.
  • Keep rest parameters at the end of the function parameter list.
  • Avoid overusing rest parameters as they may reduce function clarity.
