Call Stack

The call stack in JavaScript is like a list that keeps track of what function is currently running and what functions to run next

The call stack ensures that functions run in the correct order

How It Works:

  1. When a function is called, it gets added to the top of the stack
  2. The function runs, and when it finishes, it gets removed from the stack
  3. The program then moves to the next function in the stack
function greet() {
console.log("Hello");
sayName();
console.log("Goodbye");
}
          
function sayName() {
console.log("My name is Agney");
}
          
greet();

What Happened Step by Step:

  1. greet() is added to the stack and starts running
  2. Inside greet(), sayName() is called, so it gets added to the stack.
  3. sayName() runs, prints "My name is Agney," and then is removed from the stack
  4. Execution returns to greet(), which finishes and is removed from the stack

what is stack overflow ?

  • A stack overflow occurs when a program uses more stack memory than is allocated for it
  • The stack is a region of memory used for storing local variables, function call information, and control flow during program execution
  • When a program calls a function, information about that function (like local variables and the return address) is stored in a "stack frame" on the stack
  • If a function calls itself repeatedly without a proper base case (or stopping condition), the stack keeps growing with each call. Eventually, the stack exceeds its limit, causing a stack overflow.
  • If a function uses too much stack space (e.g., by allocating large arrays or large local variables), it may overflow the stack

Example of such code:

function introduceMe() {
console.log('Hi, My name is Agney');
introduceMe()
}
introduceMe()
console.log('Program Ended');
