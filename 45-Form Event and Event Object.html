Form Events and Event Object in JavaScript

1. Form Events in JavaScript

Form events are triggered when the user interacts with form elements (like input fields, buttons, checkboxes, etc.) in various ways. Common form events include typing into a field, submitting the form, selecting an option, or focusing on a field.

Common Form Events:

  1. `submit`

    Triggered when the form is submitted. Commonly used to validate form data before sending it to the server.

    const form = document.querySelector("form");
form.addEventListener("submit", (event) => {
    event.preventDefault(); // Prevents the form from submitting
    console.log("Form submitted!");
});
  2. `input`

    Fires whenever the user types into an input or textarea.

    const inputField = document.querySelector("input");
inputField.addEventListener("input", (event) => {
    console.log("Input value:", event.target.value);
});
  3. `change`

    Triggered when the value of an input or select element changes and the user moves focus away.

    const dropdown = document.querySelector("select");
dropdown.addEventListener("change", (event) => {
    console.log("Selected option:", event.target.value);
});
  4. `focus`

    Fires when an element (like input or textarea) gains focus (i.e., when the user clicks or tabs into it).

    const inputField = document.querySelector("input");
inputField.addEventListener("focus", () => {
    console.log("Input field is focused.");
});
  5. `blur`

    Triggered when an element loses focus.

    const inputField = document.querySelector("input");
inputField.addEventListener("blur", () => {
    console.log("Input field lost focus.");
});
  6. `reset`

    Fires when the form’s reset button is clicked, clearing all form fields.

    const form = document.querySelector("form");
form.addEventListener("reset", () => {
    console.log("Form has been reset.");
});

2. Event Object in JavaScript

The Event Object represents the event that occurred and provides information about it. It is automatically passed to event handler functions and contains useful properties and methods to handle the event.

Common Properties of the Event Object:

  1. `type`

    Specifies the type of the event (e.g., `click`, `submit`, `input`).

    document.querySelector("button").addEventListener("click", (event) => {
    console.log("Event type:", event.type); // Output: click
});
  2. `target`

    Refers to the element that triggered the event.

    document.querySelector("form").addEventListener("submit", (event) => {
    console.log("Event target:", event.target); // Output: <form>...</form>
});
  3. `currentTarget`

    Refers to the element the event listener is attached to.

    const button = document.querySelector("button");
button.addEventListener("click", (event) => {
    console.log("Current Target:", event.currentTarget); // Output: <button>...</button>
});
  4. `preventDefault()`

    Prevents the default action associated with the event.

    const form = document.querySelector("form");
form.addEventListener("submit", (event) => {
    event.preventDefault(); // Prevents form submission
    console.log("Default form submission prevented.");
});
  5. `stopPropagation()`

    Stops the event from bubbling up to parent elements.

    document.querySelector("div").addEventListener("click", () => {
    console.log("Div clicked.");
});
document.querySelector("button").addEventListener("click", (event) => {
    event.stopPropagation();
    console.log("Button clicked.");
});
  6. `key`

    Captures the key pressed during a keyboard event.

    document.addEventListener("keydown", (event) => {
    console.log("Key pressed:", event.key); // Outputs the key pressed, like 'Enter'
});
  7. `value`

    Not directly part of the Event Object but available via `event.target` for input elements.

    document.querySelector("input").addEventListener("input", (event) => {
    console.log("Current value:", event.target.value);
});

Form Event and Event Object in Action:

Here’s an example combining a form event and the event object:

<form id="myForm">
    <input type="text" id="name" placeholder="Enter your name" />
    <button type="submit">Submit</button>
</form>

<script>
    const form = document.querySelector("#myForm");
    form.addEventListener("submit", (event) => {
        event.preventDefault(); // Stop the form from reloading the page
        const nameInput = event.target.querySelector("#name").value;
        console.log("Submitted name:", nameInput);
    });
</script>

Key Points:

  • Form Events are specific to user actions on form elements.
  • Event Object provides details about the event, helping manage user interactions.
  • Use methods like `preventDefault()` to control the default behavior.
  • Access properties like `target` and `type` to customize responses to user actions.
