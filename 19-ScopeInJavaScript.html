Scope in javascript

  • In JavaScript, scope refers to the context in which variables, functions, and objects are accessible. It determines where in your code you can use or reference a particular variable or function.
    In simple words, Scope in JavaScript means where you can use a variable or function in your code

Types of Scope in JavaScript:

1. Global Scope

  • Variables or functions declared outside any function or block belong to the global scope
  • They are accessible anywhere in the code after their declaration
  • Variables outside any function or block
  • Can be used anywhere in the code
let globalVar = "I'm global";
    
function showGlobalVar() {
console.log(globalVar); // Accessible
}
            
showGlobalVar();
console.log(globalVar); // Accessible

2. Local Scope

  • Variables declared inside a function are in the local scope of that function
  • They can only be accessed within the function where they are defined
  • Variables inside a function.
  • Can only be used inside that function
function localScopeExample() {
let localVar = "I'm local";
console.log(localVar); // Accessible
}
                
localScopeExample();
console.log(localVar); // Error: localVar is not defined

3. Block Scope

  • Introduced in ES6 with let and const
  • Variables declared inside a block ( { } ) are only accessible within that block
  • Does not apply to var (it ignores block scope and is function-scoped)
  • Variables inside { } using let or const
  • Can only be used inside the { } block
{
let blockVar = "I am block-scoped";
console.log(blockVar); // Can use it here
}
                
console.log(blockVar); // Error: Cannot use it here

4. Function Scope

  • Applies to variables declared with var inside a function
  • The variable is confined to the entire function, regardless of block nesting
function functionScopeExample() {
if (true) {
var funcVar = "I'm function-scoped";
}
console.log(funcVar); // Accessible
}
                
functionScopeExample();

Lexical Scope

Lexical scope means that the scope of a variable is determined by its position in the source code. Inner functions can access variables defined in their outer functions due to this rule

function outerFunction() {
let outerVar = "Outer";
                
function innerFunction() {
console.log(outerVar); // Accessible due to lexical scope
}
                
innerFunction();
}
outerFunction();
