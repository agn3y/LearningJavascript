Arrow Functions in JavaScript

Arrow functions in JavaScript provide a shorter syntax for writing functions. They are introduced with the → syntax and are often used for functions that are small or used as arguments for other functions.

Basic Syntax

const functionName = (parameters) => {
  // function body
}

Key Features of Arrow Functions:

1. Shorter Syntax

Instead of using function keyword, you can write functions more concisely.

const add = (a, b) => a + b;

2. Implicit Return

If the function consists of a single expression, the return is implicit, so you donâ€™t need to write the return keyword.

const square = x => x * x; // Implicit return

3. No this Binding

Arrow functions do not have their own this value. Instead, they inherit the this value from the surrounding (lexical) context, which can be useful in certain situations, especially in callbacks.

const person = {
  name: "Agney",
  greet: function() {
    setTimeout(() => {
      console.log(`Hello, ${this.name}`);
    }, 1000);
  }
};
person.greet(); // Outputs: Hello, Agney

Example:

// Normal function
function multiply(a, b) {
  return a * b;
}

// Arrow function
const multiply = (a, b) => a * b;

Arrow functions are widely used for their concise syntax, especially in cases where functions are used as arguments (e.g., with map(), filter(), and reduce() methods).
