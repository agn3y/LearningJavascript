Access Parent, Sibling, and Children Elements Using JavaScript

1. Accessing Parent Element

To move upwards in the DOM hierarchy and access the parent element of an element:

Property: parentElement

Returns the immediate parent of an element.

Example:


let childElement = document.querySelector('.child');
let parentElement = childElement.parentElement; // Access parent
console.log(parentElement);

Note: If there is no parent, parentElement will return null.

2. Accessing Sibling Elements

To navigate between sibling elements at the same level in the DOM hierarchy:

Properties:

  • previousElementSibling: Access the previous sibling element.
  • nextElementSibling: Access the next sibling element.

Example:


let currentElement = document.querySelector('.current');
let previousSibling = currentElement.previousElementSibling; // Access previous sibling
let nextSibling = currentElement.nextElementSibling; // Access next sibling
console.log(previousSibling);
console.log(nextSibling);

Note: If there are no siblings in that direction, these properties return null.

3. Accessing Children Elements

To access the child elements (nested inside the current element):

Properties:

  • children: Returns an HTMLCollection (live list of all child elements).
  • firstElementChild: Access the first child element.
  • lastElementChild: Access the last child element.

Example:


let parentElement = document.querySelector('.parent');
let allChildren = parentElement.children; // Access all children
let firstChild = parentElement.firstElementChild; // Access first child
let lastChild = parentElement.lastElementChild; // Access last child

console.log(allChildren); // HTMLCollection
console.log(firstChild);
console.log(lastChild);

Note: children excludes text nodes, while childNodes includes text nodes and comments.

4. Traversing Using childNodes

childNodes: Returns a NodeList of all children, including text nodes, comments, and element nodes.

Example:


let parent = document.querySelector('.parent');
let allNodes = parent.childNodes; // Includes text and comment nodes
console.log(allNodes);

Filter Node Types: Use nodeType to distinguish between node types (1 = element, 3 = text).

5. Combining Parent, Sibling, and Child Access

You can chain these properties for advanced DOM navigation.

Example:


let grandParent = document.querySelector('.grandparent');
let firstChild = grandParent.firstElementChild; // First child
let firstChildSibling = firstChild.nextElementSibling; // Next sibling of first child
let childOfFirstChild = firstChild.firstElementChild; // Child of first child

console.log(firstChild);
console.log(firstChildSibling);
console.log(childOfFirstChild);

Best Practices

  • Use specific selectors (like querySelector) to minimize errors.
  • Always check for null values when navigating, to avoid runtime errors.
  • Be mindful of text nodes when using childNodes.
