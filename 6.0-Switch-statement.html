Understanding the Switch Statement

Syntax

switch (expression) {
    case value1:
        // Code block for value1
        break;

    case value2:
        // Code block for value2
        break;

    default:
        // Code block if no case matches
}

Key Notes

  • The break statement stops the execution of further cases in the switch block. Without it, execution will "fall through" to subsequent cases.
  • The default block is optional but serves as a fallback when no case matches the expression.
  • You can use boolean logic or boolean values in switch cases to handle specific conditions.
  • If two cases evaluate to true, the first matching case is executed, and the switch block exits (subsequent cases are ignored).

Example

const day = new Date().getDay();

switch (day) {
    case 0:
        console.log("It's Sunday!");
        break;

    case 6:
        console.log("It's Saturday!");
        break;

    default:
        console.log("It's a weekday.");
}

In this example:

  • The `day` variable holds the current day of the week (0 for Sunday, 6 for Saturday).
  • The `switch` statement checks the value of `day` and executes the corresponding case.
  • If no case matches, the default block executes.
