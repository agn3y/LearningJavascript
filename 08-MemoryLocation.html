Memory Location

Understanding how values are assigned to variables is crucial in programming. In JavaScript, values are stored in memory locations, and variables point to those locations. Below is a diagram to help visualize this concept:



The image above shows how variables store values in memory during execution. For example, when you assign a value to a variable, like the following:

        const age = 12;
        const songName = 'DanzaKuduro';

Explanation:

  1. When JavaScript runs, it allocates memory to store variables. It doesn't need to know how many variables there are in advance. It creates memory slots (addresses) to hold values as the code executes.
  2. Then the JavaScript engine assigns values to these memory slots.
  3. For primitive types (like numbers, strings, booleans), the value itself is stored in the variable. When you assign one variable to another, it copies the actual value.
  4. For objects (like arrays and functions), the variable stores a reference (memory address) to the location where the object is stored, not the object itself.
  5. When you compare objects, JavaScript checks if the memory addresses (references) are the same. Even if two objects have the same content, they might not be the same object if they are stored at different memory locations.
  6. This is also a reason why we can access a function before defining it, but with variables, this doesn't work.
