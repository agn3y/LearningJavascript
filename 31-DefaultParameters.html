Default Parameters in JavaScript

Default parameters in JavaScript allow you to set default values for function parameters. If no argument is provided for a parameter when the function is called, the default value is used. This feature was introduced in ES6 (ECMAScript 2015).

Syntax:

function functionName(param1 = defaultValue1, param2 = defaultValue2) { 
    // Function body 
}

Example:

Without Default Parameters:

function greet(name) {
    if (!name) {
        name = "Guest";
    }
    console.log(`Hello, ${name}!`);
}
greet();        // Output: Hello, Guest!
greet("John");  // Output: Hello, John!

With Default Parameters:

function greet(name = "Guest") {
    console.log(`Hello, ${name}!`);
}
greet();        // Output: Hello, Guest!
greet("John");  // Output: Hello, John!

Key Points:

Default Value: The default value is used if the argument is undefined. If you explicitly pass null, it will not use the default value.

function sayHello(name = "Guest") {
    console.log(`Hello, ${name}!`);
}
sayHello();          // Output: Hello, Guest!
sayHello(undefined); // Output: Hello, Guest!
sayHello(null);      // Output: Hello, null!

Expressions as Default Values: You can use expressions, even function calls, as default values. 

function multiply(a, b = 2) {
    return a * b;
}
console.log(multiply(5));        // Output: 10
console.log(multiply(5, 3));     // Output: 15

Order of Parameters: Default parameters should typically come after non-default ones for clarity.

function displayInfo(name, age = 18) {
    console.log(\`\${name} is \${age} years old.\`);
}
displayInfo("Alice");         // Output: Alice is 18 years old.
displayInfo("Bob", 25);       // Output: Bob is 25 years old.

Rest and Default Parameters: You can combine rest parameters with default values.

function add(a, b = 0, ...rest) {
    return a + b + rest.reduce((sum, num) => sum + num, 0);
}
console.log(add(1));             // Output: 1
console.log(add(1, 2, 3, 4));    // Output: 10

Benefits of Default Parameters:

  • Reduces the need for manual checks for undefined.
  • Makes code cleaner and more readable.
  • Avoids the need for workarounds like || for setting default values:
function greet(name) {
name = name || "Guest"; // Old workaround
console.log(\`Hello, \${name}!\`);
}
