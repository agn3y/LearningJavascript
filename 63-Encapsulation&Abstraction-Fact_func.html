Encapsulation

* Encapsulation means hiding the details of how something works and only showing what is necessary. It bundles data (like variables) and actions (like functions) together in one place, such as an object or class, and controls how they are accessed.
For example, a TV remote hides how it works inside, and you only use the buttons to control the TV.

* it means putting all things together in an object also called data hiding

Abstraction

* Abstraction means showing only the important details and hiding unnecessary complexity to make something easier to use or understand.

* it means hiding the complexity

* Using the TV remote example:

* Encapsulation is how all the buttons, circuits, and technology inside the remote are bundled together in one device. Abstraction is how you only see and use simple buttons like "Power," "Volume," or "Channel," without needing to know how the remote sends signals to the TV. In programming, abstraction works the same way: it shows only the important functionality while hiding the complex details of how it works. For example, calling a turnOn() function to power up a device hides all the underlying code handling the power mechanism.
